üß© 2. Java Architecture

| Component                      | Description                                                     |
| -------------------------------| --------------------------------------------------------------- |

üß± 3. Object-Oriented Foundations

Java is fully object-oriented, promoting modular and maintainable code.

| Principle     | Meaning                                 | Example                          |
| --------------| --------------------------------------- | -------------------------------- |
| Encapsulation | Bundle data and behavior                | Private fields + getters/setters |
| Inheritance   | Reuse properties from parent class      | `class Car extends Vehicle`      |
| Polymorphism  | One interface, multiple implementations | `draw()` in `Circle`, `Square`   |
| Abstraction   | Hiding internal logic from the user     | Abstract classes, interfaces     |

---------------------------------------------

‚ö° 4. Memory Management

Java uses automatic garbage collection to manage memory efficiently.

| Area                       | Purpose                                    |
| ---------------------------| ------------------------------------------ |
| Heap                       | Stores objects and instance variables      |
| Stack                      | Holds local variables and method calls     |
| Method Area                | Stores class metadata and static variables |
| PC Register & Native Stack | Track thread execution and native code     |

Garbage collectors like G1, ZGC, and Shenandoah handle memory cleanup automatically.

üöÄ 7. JVM Internals & Performance

* JIT (Just-In-Time Compiler): Converts hot bytecode paths to native code for faster execution.
* ClassLoader: Dynamically loads classes into memory.
* Garbage Collectors: Automatically reclaim unused objects.
* Thread Model: Supports lightweight concurrency and synchronization via `synchronized`, `Lock`, and `volatile`.
